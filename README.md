# 📚 커뮤니케이션: 미디어·디지털·학술적 리터러시 통합 교육

## 커뮤니케이션 수업 자료

미디어 리터러시, 디지털 리터러시, 학술적 리터러시의 세 영역을 통합하여 설계된 17주 과정의 커뮤니케이션 수업 자료입니다.

---

## 🎯 수업 목표

1. **미디어 리터러시**: 프레임 이론과 의사소통 이론을 이해하고 비판적 분석 능력 함양
2. **디지털 리터러시**: 바이브 코딩과 데이터 분석 기법을 활용한 실증적 분석 능력 습득
3. **학술적 리터러시**: 연구 결과를 학술적 형식으로 표현하고 발표하는 능력 개발

---

## 📋 수업 설계의 기본 방향

| 원칙 | 설명 |
|-----|------|
| **이론과 실습의 균형** | 미디어 이론을 바탕으로 실제 데이터 분석 수행 |
| **단계적 심화 학습** | 기초 이론 → 사례 분석 → 데이터 수집 → 분석 → 발표 |
| **군 특수성 반영** | 심리전 사례와 전략적 커뮤니케이션 내용 포함 |
| **비전공자 접근성 확보** | 바이브 코딩 도입으로 기술적 장벽 최소화 |

## 💻 실습 자료 (Jupyter Notebooks)

### 1. 웹 크롤링 기초 (10-11주차)

#### 📁 `fmkorea_crawler_colab_v2.ipynb`
에프엠코리아 게시물 및 댓글 크롤러

| 항목 | 내용 |
|-----|------|
| **목적** | Selenium 기반 웹 크롤링 기초 학습 |
| **주요 기능** | 키워드 검색, 날짜 필터링, 게시물/댓글 수집 |
| **출력** | Excel 파일 (게시물, 댓글) |
| **환경** | Google Colab |

**학습 포인트**:
- HTML 구조의 기초 이해
- Selenium을 활용한 동적 웹페이지 크롤링
- 봇 탐지 우회 기법

---

### 2. API 활용 데이터 수집 (12-13주차)

#### 📁 `youtube_api_kobert_analysis.ipynb`
YouTube Data API + KoBERT 감성분석

| 항목 | 내용 |
|-----|------|
| **목적** | 공식 API를 통한 안정적 데이터 수집 학습 |
| **주요 기능** | 동영상 검색, 댓글 수집, KoBERT 감성분석 |
| **출력** | CSV 파일 (분석 결과) |
| **환경** | Google Colab + YouTube API 키 |

**학습 포인트**:
- API의 개념과 인증 방식 이해
- 요청/응답 형식 학습
- API vs 크롤링의 장단점 비교

**API vs 크롤링 비교**:
| 항목 | YouTube API | 크롤링 |
|------|-------------|--------|
| 안정성 | ✅ 높음 | ⚠️ 변동 가능 |
| 속도 | ✅ 빠름 | 🐢 느림 |
| 제한 | 일일 할당량 | 차단 위험 |
| 합법성 | ✅ 공식 지원 | ⚠️ ToS 위반 가능 |

---

### 3. 감성 분석 및 시각화 (14-15주차)

#### 📁 `dcinside_kobert_analysis.ipynb`
디시인사이드 크롤링 + KoBERT 감성분석 + 혐오표현 탐지

| 항목 | 내용 |
|-----|------|
| **목적** | 종합적인 텍스트 분석 파이프라인 학습 |
| **주요 기능** | 갤러리 크롤링, 감성분석, 혐오표현 탐지, 시각화 |
| **출력** | CSV 파일, 시각화 차트 |
| **환경** | Google Colab |

**분석 파이프라인**:
```
갤러리 설정 → 게시글 수집 → 댓글 수집 → 텍스트 전처리
                                          ↓
                          KoBERT 감성분석 ← 혐오표현 탐지
                                          ↓
                                   종합 분석 및 시각화
```

---

#### 📁 `hate_speech_text_mining_education.ipynb`
혐오와 갈등 분석: 텍스트 마이닝 교육

| 항목 | 내용 |
|-----|------|
| **목적** | 혐오 표현과 갈등 패턴 탐지 방법론 학습 |
| **주요 기능** | 감성 분석, 토픽 모델링(LDA), 혐오 패턴 탐지 |
| **출력** | CSV 파일, JSON 통계, 시각화 |
| **환경** | Google Colab |

**혐오 표현 패턴 유형**:
| 유형 | 설명 |
|-----|------|
| 일반화 | 항상, 걔네, 그쪽, 다들 |
| 비하 | 한심, 멍청, 무식 |
| 선동 | 각성, 분노, 일어나 |
| 편가르기 | ~충, ~놈, 좌파, 우파 |

## 🤖 KoBERT 감성분석

모든 분석 노트북에서 사용되는 핵심 기술입니다.

| 감성 | 설명 |
|-----|------|
| **Positive (긍정)** | 긍정적인 감정 표현 |
| **Neutral (중립)** | 중립적인 표현 |
| **Negative (부정)** | 부정적인 감정 표현 |

**특징**:
- Hugging Face의 사전학습된 한국어 BERT 모델
- 신뢰도(confidence) 점수 제공
- GPU 사용 시 빠른 추론 가능

## ⚠️ 윤리적 고려사항

### 데이터 수집 시 주의사항
- **개인정보 보호법 준수**: 개인 식별 정보 수집 금지
- **저작권 및 이용약관 확인**: 각 플랫폼의 ToS 준수
- **IRB 승인**: 학술 연구 시 연구윤리위원회 승인 필요
- **데이터 2차 배포 금지**: 수집한 데이터의 무단 배포 삼가

### 분석 시 주의사항
- 과도한 크롤링은 서버에 부담을 줌
- 적절한 딜레이 유지 (3~6초 권장)
- API 일일 할당량 준수

## 🔧 문제 해결 가이드

### 크롤링 관련
| 문제 | 해결 방법 |
|-----|----------|
| 크롤링 차단 | 딜레이 증가, User-Agent 변경 |
| 페이지 로드 실패 | 대기 시간 증가, 재시도 |
| 데이터 누락 | 셀렉터 확인, HTML 구조 변경 확인 |

### API 관련
| 문제 | 해결 방법 |
|-----|----------|
| 할당량 초과 | 다음 날 대기, 프로젝트 분산 |
| 인증 오류 | API 키 확인, 권한 설정 확인 |
| 댓글 미수집 | 댓글 비활성화 동영상 확인 |

### KoBERT 관련
| 문제 | 해결 방법 |
|-----|----------|
| 메모리 부족 | 배치 크기 감소 |
| 모델 로드 실패 | 네트워크 확인, 재다운로드 |
| 한글 폰트 오류 | 나눔폰트 재설치, 런타임 재시작 |
